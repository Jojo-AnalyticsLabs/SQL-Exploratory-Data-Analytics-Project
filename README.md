# Exploratory Data Analysis (EDA) Using SQL

## Project Overview

This repository contains SQL-based Exploratory Data Analysis (EDA) projects performed on three different datasets. The analyses focus on data exploration, aggregation, trend identification, and generating meaningful business insights using SQL.

The objective of this project is to demonstrate practical SQL skills commonly used by Data Analysts to understand datasets, summarize key metrics, identify patterns, and answer business questions through data.

---

## Project Objectives

- -Explore the structure and contents of multiple datasets.
- - Understand the dimensions and characteristics of each dataset.
- - Analyze date-related trends where applicable.
- - Perform numerical analysis using SQL aggregate functions.
- - Generate meaningful business insights from the data.
- - Rank and compare top and bottom performers across different categories.

- ---

## EDA Workflow

### 1. Data Exploration
- - Inspect dataset structure
- - Review column names and data types
- - Understand the contents of each table
- - Explore missing values where applicable

- ---

### 2. Dimensions Exploration
- - Count rows and columns
- - Identify unique values
- - Explore categorical variables
- - Examine dataset composition

- ---

### 3. Date Exploration
- - Identify date ranges
- - Analyze yearly and monthly trends
- - Explore changes over time

- ---

### 4. Measure Exploration (Numerical Analysis)
Perform statistical summaries using SQL aggregate functions:

- - COUNT()
- - SUM()
- - AVG()
- - MIN()
- - MAX()

- ---

### 5. Magnitude Analysis
Measure and compare values across different categories to identify significant patterns and trends.

---

### 6. Ranking Analysis
Use SQL ranking techniques to identify:

- Top-performing categories
- - Bottom-performing categories
- - Highest and lowest values
- - Key trends and comparisons
